Android Share requires Chrome

Vivaldi cannot register Share targets on Android. It installs a home-screen wrapper, not a system WebAPK. Only Chrome (and Samsung Internet on Samsung phones) mints the native app package that adds Pick-a-Print to the Share sheet.

Installed

Install with Chrome (for Share)

  1. Remove the Vivaldi Pick-a-Print icon from your home screen.
  2. Android Settings → Apps → uninstall any Pick-a-Print entry if present.
  3. Open Chrome (not Vivaldi) → https://print.pickel.me
  4. Stay on the page ~30 seconds, tap once, then Chrome menu → Install app.
  5. Verify: Settings → Apps shows Pick-a-Print as its own app (not inside Vivaldi/Chrome).
  6. Open that app, sign in, then try Share from another app.

What Share accepts

  • Model links from Printables, Thangs, etc. → saved to library
  • .stl / .3mf files → uploaded as models
  • Photos or .zip → starts a 3D scan job

You must be signed in before sharing. Keep using Vivaldi for browsing — only the Chrome install is needed for Share.

Browser extension

Save models from Printables, MakerWorld, and other sites without opening Pick-a-Print. Your API credentials are bundled in the download.

Download extension (v{{ extension_version }}) Chrome · Edge · Brave
  1. Click Download extension above and unzip the file.
  2. Open chrome://extensions in your browser.
  3. Enable Developer mode (top right).
  4. Click Load unpacked and select the pick-a-print-extension folder.
  5. On a model page, click Save to Library or use Save current tab in the extension popup.

Chrome does not allow one-click install for self-hosted extensions — download and load unpacked is required once per browser.
